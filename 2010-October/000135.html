Outra coisa, a implementação de um "for" pode ser bastante variada. Só
para ilustrar, suponha que "para_cada()" seja um método de arrays.
Poderiamos implementar assim (pseudo-codigo):

Array:para_cada(f) {
  tamanho = this.tamanho();
  i = 0;

  [tamanho <= indice].enquanto([
    f.execute(this.indice(i));
    indice = indice + 1;
  ]);
}

E o enquanto:

Função:enquanto(f) {
  this.execute().se([ f.execute(); this.enquanto(f)], [])
}

---

...considerando que, neste pseudo-código, "this" seja sempre associado
ao receptor no contexto do método.

Seja como for, o código (e a linguagem) aqui são só ilustrativos. Eu
ainda não tenho muita sugestão de linguagem bacana em português para
representar legal esse modelo sem que seja uma "tradução" do
smalltalk...
